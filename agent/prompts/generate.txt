WHAT TO BUILD PER FORMAT CATEGORY
══════════════════════════════════════════════════

DOCUMENTS (pdf, docx, odt, rtf, pptx):
  - Render readable content — never dump raw bytes
  - PDF: render pages via canvas; include page nav (Prev/Next + page counter)
  - DOCX/ODT: render full mammoth HTML output in a styled container
  - PPTX: JSZip → extract ppt/slides/*.xml → parse and show slide text content
  - RTF: strip \rtf1 control sequences, show clean text
  - Actions: Copy Text, Download Original

SPREADSHEETS (xlsx, xls, ods, csv, tsv):
  - Render as a responsive table (cap at 500 rows for performance)
  - XLSX/ODS: show sheet selector if multiple sheets
  - Show row count and column count
  - Sticky header row, striped rows, horizontal scroll wrapper
  - Actions: Copy as CSV, Download as CSV

DATA (json, yaml, xml, toml, ini):
  - JSON: pretty-print, 2-space indent, syntax colors with Prism or manual coloring
  - YAML: parse with js-yaml, show formatted YAML
  - XML: format with indentation (DOMParser + XMLSerializer or manual)
  - TOML/INI: show with Prism syntax highlighting
  - Actions: Copy, Download Formatted; JSON also gets Download Minified

CODE (sql, graphql, proto, log):
  - Prism.js syntax highlighting with line numbers
  - Show total line count
  - Actions: Copy, Download

ARCHIVES (zip, tar, gz, bz2, xz, 7z, rar):
  - List all files with sizes in a clean table (path, size, modified date)
  - Show total: file count, total uncompressed size
  - ZIP: each row gets a Download button
  - Actions: Copy File List, Download File List as TXT

AUDIO (mp3, wav, ogg, flac, aac):
  - Create Blob from ArrayBuffer, URL.createObjectURL, render <audio controls>
  - Show filename and file size
  - No extra CDN needed

VIDEO (mp4, webm, avi, mkv, mov):
  - Create Blob, createObjectURL, render <video controls style="max-width:100%">
  - Show file size

IMAGES (webp, avif, heic, tiff, bmp, ico, gif, png, jpg, apng):
  - Display the image (convert HEIC first, decode TIFF via UTIF)
  - Show: width × height, file size
  - Actions: Download as PNG

SVG:
  - Render inline (strip <script> tags for safety)
  - "View Source" toggle button
  - Show dimensions from viewBox or width/height attributes
  - Actions: Copy SVG Source, Download SVG

MAPS/GEO (geojson, kml, gpx):
  - Render Leaflet map, auto-fit bounds
  - Show feature/point count
  - KML/GPX: convert to GeoJSON before rendering
  - Actions: Copy as GeoJSON, Download

CALENDAR (ics):
  - Parse with ical.js, show event cards
  - Card fields: title, start/end date+time, location, description
  - Show total event count
  - Actions: Copy as Text

CONTACTS (vcf):
  - Parse manually (FN, EMAIL, TEL, ORG, NOTE fields)
  - Show contact cards with name, email, phone, org
  - Avatar: circle with initials in brand color
  - Show total contact count
  - Actions: Copy as Text

EMAIL (eml):
  - Parse headers: From, To, Subject, Date
  - Render body (HTML or plain text)
  - List attachment names + sizes
  - Actions: Copy Plain Text

TEXT/SUBTITLES (txt, log, srt, vtt, ass, lrc, md):
  - TXT/LOG: monospace pre, line numbers, total line count; color ERROR/WARN/INFO lines
  - SRT/VTT/ASS: parse and show subtitle entries as a timeline table
  - LRC: parse and show lyrics with timestamps
  - Markdown: render to HTML with marked.js and a `.prose` styled container
  - Actions: Copy, Download

CERTIFICATES (pem, crt, key):
  - Use node-forge to parse; display Subject, Issuer, Valid From/To, Fingerprint
  - Actions: Copy PEM

3D (stl, glb, gltf, obj):
  - Three.js canvas with OrbitControls and basic lighting
  - Show vertex count, face count
  - Auto-rotate on load

WASM/BINARIES (wasm, exe, msi, dll, so, dylib):
  - Show file size and magic bytes identification
  - Hex dump table: offset | hex bytes | ASCII printable chars (first 256 bytes)
  - WASM: also try WebAssembly.compile and list exports

PACKAGES (deb, rpm, apk, ipa, jar, whl, gem, nupkg, crate, snap, flatpak, appimage):

  - Find and display the metadata file (MANIFEST.MF, AndroidManifest.xml, PKG-INFO, etc.)
  - Show package name, version, description

DESIGN (psd, ai, sketch, fig, xd):
  - PSD: use psd.js to render a preview
  - Others: show file size and format info; extract any readable metadata
