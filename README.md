# OmniOpener

> Open any file, right in your browser. 200+ formats, 100% client-side.

## Architecture

```
public/
├── index.html          ← SPA shell (TailwindCSS)
├── app.js              ← Client-side routing engine
├── config.json         ← Tool registry (append-only)
└── tools/
    ├── drawio-viewer.js
    ├── md-to-pdf.js
    └── ...             ← Auto-generated by the agentic loop

agent/
├── loop.sh             ← Continuous Gemini CLI generation loop
├── INSTRUCTIONS.md     ← Human override (write here, loop reads + clears)
├── queue.csv           ← Format queue
├── state.json          ← Loop progress
└── prompts/
    ├── generate.txt    ← Tool generation prompt
    ├── validate.txt    ← Validation prompt
    └── improve.txt     ← Self-improvement prompt

nginx/
└── omniopener.conf     ← Nginx site config

.github/workflows/
└── deploy.yml          ← Auto-deploy on push
```

## How It Works

1. **Routing**: `app.js` parses the URL (`/tools/pdf-opener`), finds the matching tool in `config.json`, dynamically loads the script, and calls `initTool()`.

2. **Modular Tools**: Each tool is a self-contained IIFE in `public/tools/`. Adding a new tool = adding a `.js` file + a JSON entry.

3. **Agentic Loop**: `agent/loop.sh` runs on the VM and continuously generates new tools using Gemini CLI, validates them, retries on failure, and auto-pushes.

## Adding a Tool Manually

1. Create `public/tools/my-tool.js` following the IIFE + `window.initTool` pattern
2. Add an entry to `config.json` via `jq`
3. Commit and push

## Human Override

Edit `agent/INSTRUCTIONS.md` with any instruction. The loop reads it on every iteration and acts on it.

## VM Setup

```bash
./scripts/install-agent.sh
```
